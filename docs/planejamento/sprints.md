# Planejamento Incremental por Sprint - ReservaIF 2.0

Este documento detalha a evolução do projeto através dos incrementos realizados em cada Sprint.

---

## Sprint 1: Concepção e Visão
- **Objetivo:** Definir o escopo do projeto e os papéis da equipe.
- **Incremento:** Documento de visão, identificação do problema e objetivos principais.
- **Status:** Concluído.

## Sprint 2: Requisitos e Prototipação Inicial
- **Objetivo:** Detalhar as primeiras Histórias de Usuário e criar a identidade visual.
- **Incremento:** 
    - Backlog inicial (HU01 a HU05).
    - Protótipos de baixa fidelidade (Login e Dashboard).
    - Definição dos Atores e Regras de Negócio básicas.
- **Status:** Concluído.

## Sprint 3: Modelagem Técnica e Manutenção
- **Objetivo:** Estruturar o banco de dados e planejar as funções de manutenção.
- **Incremento:**
    - Dicionário de Dados e Script SQL (DDL).
    - Planejamento das HUs de Manutenção (HU06, HU07, HU10).
    - Definição da Arquitetura do Sistema (MVC).
- **Status:** Concluído.

## Sprint 4: Detalhamento e Melhoria de UX
- **Objetivo:** Refinar a experiência do usuário e criar relatórios estratégicos.
- **Incremento:**
    - Mapeamento completo das 16 Histórias de Usuário.
    - Planejamento de notificações e histórico de reservas.
    - Estruturação final do repositório conforme padrões acadêmicos.
- **Status:** Em andamento (Fase de documentação final).
